## 用例详细说明

### 核心用例（P0 - 必须实现）

| 用例编号 | 用例名称 | 说明 | 参与者 |
|---------|---------|------|--------|
| UC-01 | 文本输入食物及克重 | 用户输入食物名称、食材、重量等信息 | 用户 |
| UC-02 | AI估算热量及营养 | 调用大模型API分析食物热量、蛋白质、碳水、脂肪 | AI服务 |
| UC-03 | AI生成食物图标 | 根据食物名称调用图像生成模型生成图标 | AI服务 |
| UC-04 | 保存食物记录 | 将食物记录保存到本地Room数据库 | 用户 |
| UC-05 | 查看今日摄入 | 首页展示当天按餐次分类的摄入记录 | 用户 |
| UC-06 | 设置每日热量目标 | 设置每日热量摄入上限 | 用户 |

## 关键业务流程

### 流程1：文本输入记录食物
```
用户 ──► 打开App ──► 点击"+" ──► 选择"文本输入" 
  │
  ▼
输入食物描述 ──► 点击"估算热量" ──► 调用AI API
  │
  ▼
AI返回热量数据 ──► AI生成食物图标 ──► 显示结果页面
  │
  ▼
用户确认 ──► 保存到本地数据库 ──► 首页更新今日摄入
```

### 流程2：拍照记录包装食品
```
用户 ──► 点击"+" ──► 选择"拍照识别" ──► 拍摄营养表
  │
  ▼
AI识别营养成分 ──► 解析热量数据 ──► 使用照片作为图标
  │
  ▼
显示结果页面 ──► 用户确认/修改 ──► 保存记录
```

### 流程3：饭点提醒流程（OPPO流体云）
```
系统 ──► 定时触发（8:00/12:00/18:00）──► 发送OPPO流体云通知
  │
  ▼
用户点击通知 ──► 直达App录入页面 ──► 快速记录食物
  │
  ▼
保存记录 ──► 更新今日摄入 ──► 完成
```

## 用户场景示例

### 场景1：记录午餐（文本输入）
```
1. 用户打开App，看到今日已摄入850千卡/目标2000千卡
2. 点击右下角"+"按钮
3. 选择"文本输入"
4. 输入："番茄炒蛋，番茄150g，鸡蛋2个，油10g，米饭200g"
5. 点击"估算热量"
6. App调用AI API：
   - GPT-4o分析热量：约520千卡
   - DALL-E生成番茄炒蛋图标
7. 显示结果：
   - 图标：[AI生成的番茄炒蛋图]
   - 总热量：520千卡
   - 蛋白质：18g，碳水：65g，脂肪：20g
   - 食材明细列表
8. 用户确认后保存
9. 首页更新：今日摄入 1370千卡/2000千卡
```

### 场景2：记录零食（拍照识别）
```
1. 用户吃了一包薯片，想记录热量
2. 点击"+"按钮，选择"拍照识别"
3. 拍摄薯片包装的营养成分表
4. AI识别出：每100g含热量2500千焦（约598千卡）
5. 用户输入食用量：35g
6. 计算：598 × 0.35 = 209千卡
7. 使用营养成分表照片作为图标
8. 保存记录
```

### 场景3：饭点提醒响应
```
1. 12:00 手机收到OPPO流体云通知：
   "午餐时间到了，今日还可摄入1150千卡"
2. 用户点击通知
3. 直接打开App的食物录入页面
4. 快速记录午餐
5. 完成记录
```

### 场景4：查看本周统计
```
1. 用户切换到"统计"页面
2. 查看本周热量趋势柱状图
3. 发现周三摄入超标（2800千卡）
4. 点击查看详情，看到当晚吃了火锅
5. 查看营养成分饼图：脂肪占比45%偏高
6. 决定下周控制高脂食物摄入
```

## 数据存储说明

### 本地存储（Room数据库）
```
所有数据存储在本地，无需服务器：
├── 食物记录表（FoodRecord）
├── 用户设置表（UserSettings）
├── 图标缓存表（IconCache）
└── 备份文件（JSON/CSV导出）
```

### 图标存储策略
```
文本/语音输入 ──► AI生成图标 ──► 本地缓存
拍照输入 ──► 使用原照片 ──► 本地压缩存储
```
