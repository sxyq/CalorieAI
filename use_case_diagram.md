## 用例详细说明

### 核心用例（P0 - 必须实现）

| 用例编号 | 用例名称 | 说明 | 参与者 |
|---------|---------|------|--------|
| UC-01 | 文本输入食物及克重 | 用户输入食物名称、食材、重量等信息 | 用户 |
| UC-02 | AI估算热量及营养 | 调用大模型API分析食物热量、蛋白质、碳水、脂肪 | AI服务 |
| UC-03 | AI生成食物图标 | 根据食物名称调用图像生成模型生成图标 | AI服务 |
| UC-04 | 保存食物记录 | 将食物记录保存到本地Room数据库 | 用户 |
| UC-05 | 查看今日摄入 | 首页展示当天按餐次分类的摄入记录 | 用户 |
| UC-06 | 设置每日热量目标 | 设置每日热量摄入上限 | 用户 |
| UC-07 | 日期切换查看 | 前天/昨天/今天日期切换查看历史记录 | 用户 |

## 用户场景示例

### 场景1：记录午餐（文本输入）
```
1. 用户打开App，看到今日已摄入850千卡/目标2000千卡
2. 点击右下角"+"按钮
3. 选择"文本输入"
4. 输入："番茄炒蛋，番茄150g，鸡蛋2个，油10g，米饭200g"
5. 点击"估算热量"
6. App调用AI API：
   - GPT-4o分析热量：约520千卡
   - DALL-E生成番茄炒蛋图标
7. 显示结果：
   - 图标：[AI生成的番茄炒蛋图]
   - 总热量：520千卡
   - 蛋白质：12g，碳水：15g，脂肪：18g
   - 食材明细列表
8. 用户确认后保存
9. 首页更新：今日摄入 1370千卡/2000千卡
```

### 场景2：查看昨天记录
```
1. 用户在首页顶部日期栏向左滑动
2. 从"今天"切换到"昨天"（丝滑动效）
3. 页面平滑过渡到昨天的记录列表
4. 查看昨天摄入的食物：
   - 早餐：豆浆+油条 450千卡
   - 午餐：番茄炒蛋+米饭 520千卡
   - 晚餐：火锅 800千卡
5. 总摄入：1770千卡
```

### 场景3：配置AI API
```
1. 用户点击右上角三个点按钮
2. 选择"设置"
3. 进入设置页面，看到分组卡片式布局
4. 点击"CalorieAI AI"卡片
5. 进入AI配置页面
6. 选择OpenAI标签
7. 输入API Key
8. 选择模型：GPT-4o
9. 设置温度参数：0.7
10. 设置最大Token：2000
11. 自定义系统提示词
12. 保存配置
13. 点击"Token使用统计"查看使用情况
```

### 场景4：使用概览功能
```
1. 用户点击右上角三个点按钮
2. 选择"概览"
3. 进入统计页面
4. 查看今日统计：
   - 已摄入：1250千卡
   - 目标：2000千卡
   - 剩余：750千卡
5. 切换到"本周"标签
6. 查看本周热量趋势柱状图
```
