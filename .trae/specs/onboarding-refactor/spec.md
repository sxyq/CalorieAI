# CalorieAI 全面重构与功能增强规格文档

## 变更ID
onboarding-refactor

## 变更原因
当前应用存在以下问题需要解决：
1. 新用户缺乏引导流程，首次使用体验差
2. 概览页面上月总结模块数据呈现不够直观
3. AI助手回复格式混乱，缺乏打字机效果，用户体验不佳
4. 历史对话管理界面UI陈旧，功能不完善
5. 体重统计缺乏AI预测功能
6. 底部导航结构不清晰，功能入口分散

## 变更内容

### 1. 新用户引导流程（6个步骤）
**BREAKING**: 新增首次启动引导流程，影响应用启动逻辑

#### 引导界面一：基本信息采集
- 性别选择（男/女/其他）
- 出生日期选择（年份+月份选择器）
- 表单验证（必填项标记、提交前验证）
- 本地临时存储

#### 引导界面二：身体数据采集
- 体重输入（30-200kg范围验证）
- 身高输入（100-250cm范围验证）
- 单位选择（kg/cm）
- 实时校验和错误提示

#### 引导界面三：生活习惯设置
- 运动习惯选择（复用个人信息编辑模块）
- 每日热量目标选择（四个预设按钮）
- 状态保存和传递

#### 引导界面四：目标设定
- 目标选择（减脂/增肌/增重/保持现状）
- 个人信息管理界面同步添加目标选择
- AI助手提示词模板集成目标字段
- 数据双向同步

#### 引导界面五：目标细化
- 目标体重输入
- 减肥策略选择（激进/平和/温和）
- 概览页面体重统计区域显示目标体重和策略标签
- "重新生成曲线"按钮功能

#### 引导界面六：完成确认
- 参考设计实现
- 视觉风格统一
- 完成确认和跳转逻辑

### 2. 概览页面上月总结重构
- GitHub贡献热力图样式可视化
- 7×4/5日期网格布局
- 5级颜色梯度表示打卡频率
- 悬停显示详细数据（日期、打卡次数、完成率）
- 自适应不同屏幕尺寸

### 3. AI助手优化
#### 3.1 Markdown格式配置
- 回复控制在300字以内
- 添加表情符号增强可读性
- 简洁精炼的内容风格

#### 3.2 Markdown渲染组件重构
- 行高1.5倍
- 代码块语法高亮
- 列表缩进（一级2em，二级4em）
- 移动端和桌面端适配

### 4. AI助手打字机效果
- 逐字显示动态效果
- 打字速度：中文字符50-80字/分钟
- 参考LongCat官方文档实现流式接收
- 断句和标点符号正确处理
- 加载状态指示和异常中断处理

### 5. 历史对话管理界面重构
- 卡片式对话列表布局
- 用户/AI消息气泡样式区分
- 相对时间格式时间戳
- 搜索框和高亮匹配结果
- 日期、类型筛选功能
- 标签分类功能
- 响应式设计

### 6. AI助手形态过渡动画
- 形态切换动画（300ms，ease-in-out）
- 状态提示文字
- 60fps性能保证
- 低配置设备降级方案

### 7. 体重统计AI预测
- 折线图添加AI预测曲线（虚线）
- 实际数据曲线（实线）对比
- 调用AI五次取平均值
- 预测结果缓存机制
- 生成/更新/清除控制逻辑

### 8. UI全面重构
**BREAKING**: 移除原有导航结构，改为底部导航栏

#### 底部导航栏（从左到右）
1. **概览入口** - 显示概览页面
2. **功能入口** - 食物热量手动记录和AI导入
3. **身体档案** - 整合个人信息编辑、体重记录、身高记录、运动热量记录
4. **我的界面** - 整合设置及其他内容

#### 导航栏特性
- 所有页面固定显示
- 当前页面对应项高亮
- 点击状态反馈动画

### 受影响模块
- 应用启动流程

- 概览页面
- AI助手模块

- 体重统计模块

- Markdown渲染组件

- AI预测服务

## 技术要求

### 性能要求
- 动画帧率保持60fps以上
- 低配置设备提供降级方案
- AI预测缓存减少重复调用

### 兼容性要求
- 移动端和桌面端适配
- 不同屏幕尺寸自适应
- 支持Android 8.0+

### 数据要求
- 引导数据本地临时存储
- 用户目标数据持久化
- AI预测结果缓存24小时
