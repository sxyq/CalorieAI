# AI食物热量记录App - 需求文档

## 1. 项目概述

### 1.1 项目名称
**CalorieAI** - AI驱动的智能食物热量记录助手

### 1.2 核心定位
一款简洁高效的食物热量记录App，通过AI大模型分析用户输入的食物食材及克重，智能估算热量和营养成分，帮助用户管理每日饮食摄入。

### 1.3 设计参考
界面风格、操作习惯参考 [Deadliner](https://github.com/AritxOnly/Deadliner) 项目：
- 简洁现代的Material3设计风格
- 流畅的滑动手势操作
- 丰富的动画效果
- 直观的数据展示

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 食物录入
| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 文本输入 | 输入食物名称、食材、克重，如"番茄炒蛋，番茄150g，鸡蛋2个，油10g" | P0 |
| 语音输入 | 语音转文字输入食物信息 | P1 |
| 拍照识别 | 拍摄包装食品的营养成分表进行识别 | P1 |

#### 2.1.2 AI热量估算
| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 智能分析 | 调用大模型API分析食物热量、蛋白质、碳水、脂肪 | P0 |
| 食材拆解 | 显示各食材的热量贡献明细 | P1 |
| 置信度标注 | 标注AI估算的置信度（高/中/低） | P1 |

#### 2.1.3 记录管理
| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 保存记录 | 将食物记录保存到本地数据库 | P0 |
| 查看历史 | 按日期查看所有已保存的记录 | P0 |
| 编辑记录 | 修改已保存的食物记录 | P1 |
| 删除记录 | 删除不需要的记录（左滑删除+确认） | P1 |
| 批量操作 | 多选模式：批量删除、加星 | P2 |
| 分类展示 | 按餐次分类（早餐/午餐/晚餐/加餐） | P1 |

### 2.2 界面与交互（参考Deadliner）

#### 2.2.1 首页设计
- **今日摄入概览**：顶部显示当天总热量、目标热量、剩余可摄入
- **餐次分类展示**：按早餐/午餐/晚餐/加餐分组显示记录
- **快速添加**：右下角悬浮按钮（+）快速添加食物
- **空状态提示**：无记录时显示引导文案

#### 2.2.2 滑动手势
- **右滑**：标记为收藏/加星（可选动画效果）
- **左滑**：删除并弹出确认对话框
- **长按**：进入多选模式

#### 2.2.3 筛选与搜索
- **搜索**：按食物名称搜索历史记录
- **筛选**：按日期范围、餐次类型筛选
- **排序**：按时间、热量排序

#### 2.2.4 数据统计
- **今日统计**：实时显示当日摄入数据
- **周/月报表**：柱状图/折线图展示热量趋势
- **营养分析**：蛋白质/碳水/脂肪占比饼图

### 2.3 智能图标生成

| 场景 | 处理方式 | 优先级 |
|-----|---------|--------|
| 文本/语音输入 | 调用图像生成模型（如DALL-E/Stable Diffusion）根据食物名称实时生成图标 | P1 |
| 拍照输入 | 使用拍摄的营养成分表照片作为图标 | P0 |
| 缓存机制 | 生成的图标本地缓存，避免重复生成 | P1 |

### 2.4 饭点后台通知（OPPO流体云）

#### 2.4.1 通知场景
- **早餐提醒**：8:00 提醒记录早餐
- **午餐提醒**：12:00 提醒记录午餐
- **晚餐提醒**：18:00 提醒记录晚餐
- **加餐提醒**：根据用户习惯智能提醒

#### 2.4.2 OPPO流体云集成
参考 [阿里云OPPO流体云文档](https://help.aliyun.com/zh/document_detail/2997310.html)：
- **场景类型**：TASK（任务提醒）
- **展示内容**：当前餐次、建议摄入热量、快速记录入口
- **交互**：点击直达App录入页面

#### 2.4.3 通知配置
- 可自定义提醒时间
- 可开关各类提醒
- 勿扰模式适配

### 2.5 桌面小组件

| 类型 | 功能 | 优先级 |

| 餐次提醒 | 显示下一餐建议摄入 | P2 |

## 3. 非功能需求

### 3.1 数据存储
- **本地存储**：使用Room数据库存储所有数据
- **无需服务器**：所有数据保存在用户设备本地
- **数据安全**：敏感数据加密存储

### 3.2 性能要求
- **启动时间**：冷启动 < 2秒
- **AI响应**：热量估算 < 3秒
- **列表流畅度**：60fps滚动体验

### 3.3 离线支持
- **完全离线可用**：除AI估算外，其他功能离线可用
- **离线缓存**：AI结果本地缓存，支持离线查看
- **联网后同步**：离线记录联网后自动同步AI结果

### 3.4 兼容性
- **最低API**：Android 12 (API 31)
- **目标API**：Android 14 (API 34)
- **设备适配**：支持手机、平板

## 5. 用户场景

### 场景1：记录午餐
```
1. 用户点击右下角"+"按钮
2. 选择"文本输入"
3. 输入："番茄炒蛋，番茄150g，鸡蛋2个，油10g"
4. 点击"估算热量"
5. App调用AI API分析
6. 显示结果：
   - 生成食物图标（AI图像生成）
   - 总热量：约280千卡
   - 蛋白质：12g，碳水：15g，脂肪：18g
   - 食材明细列表
7. 用户确认后保存记录
8. 首页显示今日摄入更新
```

### 场景2：拍照记录包装食品
```
1. 用户点击右下角"+"按钮
2. 选择"拍照识别"
3. 拍摄食品包装的营养成分表
4. AI识别营养成分数据
5. 用户确认或修改数据
6. 保存记录（使用照片作为图标）
7. 首页显示更新
```

### 场景3：饭点提醒
```
1. 12:00 OPPO流体云推送午餐提醒
2. 用户点击通知
3. 直达App食物录入页面
4. 用户快速记录午餐
```

### 场景4：查看统计
```
1. 用户切换到"统计"页面
2. 查看本周热量趋势图
3. 查看营养成分占比饼图
4. 查看各餐次热量分布
```

## 8. 风险与应对

| 风险 | 影响 | 应对措施 |
|-----|------|---------|
| AI估算不准确 | 高 | 提供置信度标注，允许用户手动修正 |
| AI API成本高 | 中 | 本地缓存结果，限制免费用户调用次数 |
| 图标生成慢 | 中 | 异步生成，先显示占位图 |
| 离线功能受限 | 低 | 明确提示用户，缓存常用食物数据 |
