# AI食物热量记录App - 需求文档

## 1. 项目概述

### 1.1 项目名称
**CalorieAI** - AI驱动的智能食物热量记录助手

### 1.2 核心定位
一款简洁高效的食物热量记录App，通过AI大模型分析用户输入的食物食材及克重，智能估算热量和营养成分，帮助用户管理每日饮食摄入。

### 1.3 设计参考
界面风格、操作习惯参考 [Deadliner](https://github.com/AritxOnly/Deadliner) 项目：
- 简洁现代的Material3 Expressive设计风格
- 流畅的滑动手势操作
- 丰富的动画效果
- 直观的数据展示
- 顶部日期切换栏（前天/昨天/今天）
- 右上角三个点菜单弹窗

## 2. 功能需求

| 功能 | 描述 | 状态 |

| 拍照识别 | 拍摄食物照片或包装食品的营养成分表进行识别 | ✅ 已实现 |

#### 2.1.2 AI热量估算
| 功能 | 描述 | 状态 |

## 4. 非功能需求

### 4.1 数据存储
- **本地存储**：使用Room数据库存储所有数据 ✅
- **无需服务器**：所有数据保存在用户设备本地 ✅
- **数据安全**：敏感数据加密存储 ✅

### 4.2 性能要求
- **启动时间**：冷启动 < 2秒 ✅
- **AI响应**：热量估算 < 3秒 ✅
- **列表流畅度**：60fps滚动体验 ✅

### 4.3 离线支持
- **完全离线可用**：除AI估算外，其他功能离线可用 ✅
- **离线缓存**：AI结果本地缓存，支持离线查看 ✅
- **联网后同步**：离线记录联网后自动同步AI结果 ✅

### 4.4 兼容性
- **最低API**：Android 8.0 (API 26) ✅
- **目标API**：Android 14 (API 34) ✅
- **设备适配**：支持手机、平板 ✅

## 7. 用户场景

### 场景1：记录午餐（文本输入）
```
1. 用户打开App，看到今日已摄入850千卡/目标2000千卡
2. 点击右下角"+"按钮
3. 选择"文本输入"
4. 输入："番茄炒蛋，番茄150g，鸡蛋2个，油10g，米饭200g"
5. 点击"估算热量"
6. App调用AI API：
   - GPT-4o分析热量：约520千卡
7. 显示结果：
   - 总热量：520千卡
   - 蛋白质：12g，碳水：15g，脂肪：18g
   - 食材明细列表
8. 用户确认后保存
9. 首页更新：今日摄入 1370千卡/2000千卡
```

### 场景2：拍照记录包装食品
```
1. 用户吃了一包薯片，想记录热量
2. 点击"+"按钮，选择"拍照识别"
3. 拍摄薯片包装的营养成分表
4. AI识别出：每100g含热量2500千焦（约598千卡）
5. 用户输入食用量：35g
6. 计算：598 × 0.35 = 209千卡
7. 保存记录
```

### 场景3：使用AI营养助手
```
1. 用户在首页看到悬浮AI助手球
2. 点击打开迷你对话窗口
3. 询问"我今天吃了什么？"
4. AI回复今日饮食总结
5. 用户发送一张食物照片
6. AI分析照片中的食物并提供营养建议
7. 点击展开按钮进入全屏对话模式
8. 获得更详细的营养分析和建议
```

### 场景4：记录运动消耗
```
1. 用户完成30分钟跑步
2. 在首页点击"+"按钮
3. 选择"记录运动"
4. 选择运动类型：跑步
5. 输入时长：30分钟
6. 系统自动计算消耗热量：约300千卡
7. 保存记录
8. 首页显示今日净摄入（摄入-消耗）更新
```

### 场景5：查看数据统计
```
1. 用户切换到"统计"页面
2. 查看本周热量趋势折线图
3. 查看营养成分分布饼图
4. 查看各餐次热量对比柱状图
5. 查看营养素摄入雷达图
6. 发现蛋白质摄入不足
7. 决定调整明日饮食计划
```

### 场景6：切换日期查看历史
```
1. 用户在首页顶部日期栏向左滑动
2. 从"今天"切换到"昨天"
3. 页面丝滑过渡到昨天的记录
4. 查看昨天摄入的食物列表
5. 长按某条记录进行编辑
```

### 场景7：通过菜单访问设置
```
1. 用户点击右上角三个点按钮
2. 弹出菜单：设置、概览、编辑资料
```
