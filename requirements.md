# AI食物热量记录App - 需求文档

## 1. 项目概述

### 1.1 项目名称
**CalorieAI** - AI驱动的智能食物热量记录助手

### 1.2 核心定位
一款简洁高效的食物热量记录App，通过AI大模型分析用户输入的食物食材及克重，智能估算热量和营养成分，帮助用户管理每日饮食摄入。

### 1.3 设计参考
界面风格、操作习惯参考 [Deadliner](https://github.com/AritxOnly/Deadliner) 项目：
- 简洁现代的Material3 Expressive设计风格
- 流畅的滑动手势操作
- 丰富的动画效果
- 直观的数据展示
- 顶部日期切换栏（前天/昨天/今天）
- 右上角三个点菜单弹窗

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 食物录入
| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 文本输入 | 输入食物名称、食材、克重，如"番茄炒蛋，番茄150g，鸡蛋2个，油10g" | P0 |
| 语音输入 | 语音转文字输入食物信息 | P1 |
| 拍照识别 | 拍摄包装食品的营养成分表进行识别 | P1 |

#### 2.1.2 AI热量估算
| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 智能分析 | 调用大模型API分析食物热量、蛋白质、碳水、脂肪 | P0 |
| 食材拆解 | 显示各食材的热量贡献明细 | P1 |
| 置信度标注 | 标注AI估算的置信度（高/中/低） | P1 |

#### 2.1.3 记录管理
| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 保存记录 | 将食物记录保存到本地数据库 | P0 |
| 查看历史 | 按日期查看所有已保存的记录 | P0 |
| 编辑记录 | 修改已保存的食物记录 | P1 |
| 删除记录 | 删除不需要的记录（左滑删除+确认） | P1 |
| 批量操作 | 多选模式：批量删除、加星 | P2 |
| 分类展示 | 按餐次分类（早餐/午餐/晚餐/加餐） | P1 |

### 2.2 界面与交互（参考Deadliner）

#### 2.2.1 首页设计
- **顶部日期切换栏**：显示前天/昨天/今天，支持左右滑动切换，丝滑动画
- **右上角菜单**：三个点按钮，点击弹出设置/概览/编辑资料选项
- **今日摄入概览**：显示当天总热量、目标热量、剩余可摄入
- **餐次分类展示**：按早餐/午餐/晚餐/加餐分组显示记录
- **快速添加**：右下角悬浮按钮（+）快速添加食物
- **空状态提示**：无记录时显示引导文案

#### 2.2.2 滑动手势
- **右滑**：标记为收藏/加星（可选动画效果）
- **左滑**：删除并弹出确认对话框
- **长按**：进入多选模式

#### 2.2.3 筛选与搜索
- **搜索**：按食物名称搜索历史记录
- **筛选**：按日期范围、餐次类型筛选
- **排序**：按时间、热量排序

#### 2.2.4 数据统计
- **今日统计**：实时显示当日摄入数据
- **周/月报表**：柱状图/折线图展示热量趋势
- **营养分析**：蛋白质/碳水/脂肪占比饼图

### 2.3 智能图标生成

| 场景 | 处理方式 | 优先级 |
|-----|---------|--------|
| 文本/语音输入 | 调用图像生成模型（如DALL-E/Stable Diffusion）根据食物名称实时生成图标 | P1 |
| 拍照输入 | 使用拍摄的营养成分表照片作为图标 | P0 |
| 缓存机制 | 生成的图标本地缓存，避免重复生成 | P1 |

### 2.4 饭点后台通知（OPPO流体云）

#### 2.4.1 通知场景
- **早餐提醒**：8:00 提醒记录早餐
- **午餐提醒**：12:00 提醒记录午餐
- **晚餐提醒**：18:00 提醒记录晚餐
- **加餐提醒**：根据用户习惯智能提醒

#### 2.4.2 OPPO流体云集成
参考 [阿里云OPPO流体云文档](https://help.aliyun.com/zh/document_detail/2997310.html)：
- **场景类型**：TASK（任务提醒）
- **展示内容**：当前餐次、建议摄入热量、快速记录入口
- **交互**：点击直达App录入页面

#### 2.4.3 通知配置
- 可自定义提醒时间
- 可开关各类提醒
- 勿扰模式适配

### 2.5 桌面小组件

| 餐次提醒 | 显示下一餐建议摄入 | P2 |

## 4. 非功能需求

### 4.1 数据存储
- **本地存储**：使用Room数据库存储所有数据
- **无需服务器**：所有数据保存在用户设备本地
- **数据安全**：敏感数据加密存储

### 4.2 性能要求
- **启动时间**：冷启动 < 2秒
- **AI响应**：热量估算 < 3秒
- **列表流畅度**：60fps滚动体验

### 4.3 离线支持
- **完全离线可用**：除AI估算外，其他功能离线可用
- **离线缓存**：AI结果本地缓存，支持离线查看
- **联网后同步**：离线记录联网后自动同步AI结果

### 4.4 兼容性
- **最低API**：Android 8.0 (API 26)
- **目标API**：Android 14 (API 34)
- **设备适配**：支持手机、平板

## 7. 用户场景

### 场景1：记录午餐（文本输入）
```
1. 用户打开App，看到今日已摄入850千卡/目标2000千卡
2. 点击右下角"+"按钮
3. 选择"文本输入"
4. 输入："番茄炒蛋，番茄150g，鸡蛋2个，油10g，米饭200g"
5. 点击"估算热量"
6. App调用AI API：
   - GPT-4o分析热量：约520千卡
   - DALL-E生成番茄炒蛋图标
7. 显示结果：
   - 图标：[AI生成的番茄炒蛋图]
   - 总热量：520千卡
   - 蛋白质：12g，碳水：15g，脂肪：18g
   - 食材明细列表
8. 用户确认后保存
9. 首页更新：今日摄入 1370千卡/2000千卡
```

### 场景2：拍照记录包装食品
```
1. 用户吃了一包薯片，想记录热量
2. 点击"+"按钮，选择"拍照识别"
3. 拍摄薯片包装的营养成分表
4. AI识别出：每100g含热量2500千焦（约598千卡）
5. 用户输入食用量：35g
6. 计算：598 × 0.35 = 209千卡
7. 使用营养成分表照片作为图标
8. 保存记录
```

### 场景3：饭点提醒响应
```
1. 12:00 手机收到OPPO流体云通知：
   "午餐时间到了，今日还可摄入1150千卡"
2. 用户点击通知
3. 直接打开App的食物录入页面
4. 快速记录午餐
5. 完成记录
```

### 场景4：查看本周统计
```
1. 用户切换到"统计"页面
2. 查看本周热量趋势柱状图
3. 发现周三摄入超标（2800千卡）
4. 点击查看详情，看到当晚吃了火锅
5. 查看营养成分饼图：脂肪占比45%偏高
6. 决定下周控制高脂食物摄入
```

### 场景5：切换日期查看历史
```
1. 用户在首页顶部日期栏向左滑动
2. 从"今天"切换到"昨天"
3. 页面丝滑过渡到昨天的记录
4. 查看昨天摄入的食物列表
5. 长按某条记录进行编辑
```

### 场景6：通过菜单访问设置
```
1. 用户点击右上角三个点按钮
2. 弹出菜单：设置、概览、编辑资料
3. 选择"设置"
4. 进入设置页面，分组卡片式布局
5. 点击"CalorieAI AI"进行API配置
```

## 9. 风险与应对

| 风险 | 影响 | 应对措施 |
|-----|------|---------|
| AI估算不准确 | 高 | 提供置信度标注，允许用户手动修正 |
| AI API成本高 | 中 | 本地缓存结果，限制免费用户调用次数 |
| 图标生成慢 | 中 | 异步生成，先显示占位图 |
| 离线功能受限 | 低 | 明确提示用户，缓存常用食物数据 |
| UI适配复杂 | 中 | 参考Deadliner源码，分步实现 |
